## 背景

管理员需要在处理用户问题时，直接查看某个用户从过去到现在的预约情况。当前系统只有全局预约导出，适合批量出表，不适合按人排查。

## 目标

- 在用户管理中直接查看某个用户的预约历史。
- 支持从最近记录一直追溯到更早记录。
- 明确标识每条预约的状态，方便判断是否爽约、是否签到。

## 方案

### 入口

将预约历史放进管理员用户详情弹窗，不新增独立页面。

### 数据

- 预约历史来自 `reservations`
- 座位号来自 `seats`
- 是否已签到通过匹配 `checkins(user_id, seat_id, check_date)` 判断

### 展示

- 默认加载最近 50 条
- 支持“加载更多”继续查看更早记录
- 每条记录展示：
  - 预约日期
  - 座位号
  - 创建时间
  - 状态标签

### 状态规则

- `已签到`：存在同用户、同座位、同日期签到记录
- `已取消`：预约状态不是 `active`
- `已过期未签到`：预约日期早于今天且没有签到
- `生效中`：今天或未来的有效预约且未签到

## 风险控制

- 只在管理员查看用户详情时按需请求，不影响普通页面。
- 采用分页，避免一次性加载单个用户的全部历史导致弹窗过重。

## 验证

- 管理员打开用户详情时能看到最近预约历史
- 点击“加载更多”可以继续向前追溯
- 状态标签与实际签到/取消情况一致
